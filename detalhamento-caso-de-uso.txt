Detalhamento requisito
	- Vende Produto
		Pré-condições
			- Usuário autenticado no sistema com os seguintes perfis:
				Administrador ou Atendente
			- Deve existir produtos em estoque
		Eventos
			1. O caso de uso se inicia quando o operador recebe a solicitação de compra de um ou mais produtos
			2. O operador informa os dados da venda
			3. O operador solicita a venda do(s) produto(s)
			4. O sistema débita a quantidade comprada por produto do estoque
			5. O sistema registra a venda
			6. O caso de uso é encerrado
		Pós-condições
			- A quantidade de items vendidos por produto é débitado do estoque
			- A venda é registrada
		Fluxos Alternativos
			Fluxo alternativo produto sem estoque
			Pré-condições
				1. No passo 1 do fluxo principal
			Passos	
				1. O sistema apresenta apresenta uma mensagem que não há produtos em estoque
	- Cadastra Produto
		Pré-condições
			- Usuário autenticado no sistema como administrador
		Eventos
			1. O caso de uso se inicia quando o operador solicita o cadastro de produto
			2. O operador informa as informações sobre o produto
			3. O operador solicita o cadastro do produto
			4. O sistema cadastra o produto
			5. O caso de uso é encerrado
		Pós-condições
			- O produto é salvo no sistema
	- Cadastra Estoque
		Pré-condições
			- Usuário autenticado no sistema como administrador
		Eventos
			1. O caso de uso se inicia quando o operador recebe novos produtos
			2. O operador associa as informações do novo produto em estoque
			3. O operador solicita que o estoque seja criado
			4. O sistema cadastra o estoque
			5. O caso de uso é encerrado
		Pós-condições
			- O estoque é salvo no sistema
		Fluxos Alternativos
			Fluxo alternativo nenhum produto cadastrado
			Pré-condições
				1. No passo 1 do fluxo principal
			Passos	
				1. O sistema apresenta apresenta uma mensagem que não há produtos cadastrados
